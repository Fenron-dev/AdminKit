    Schnellstart
    
                
      
                  
    1. Session erstellen — oben rechts auf „+ Session" klicken, Namen vergeben (z.B. Kundenname + Datum).
      2. 
                  
    2. Vollständiger Scan — Dashboard → „Vollständiger Scan" startet alle Scanner sequentiell.
      3. 
                  
    3. Einzelscans — In jedem Tab gibt es einen ↺-Button für Einzel-Scans.
      4. 
                  
    4. Export — Einstellungen → Export → HTML- oder CSV-Report erstellen.
      5. 
                
    
                
    Hinweis: Alle Scans laufen lokal auf dem Gerät. Es werden keine Daten an externe Server übertragen (außer bei VirusTotal/KI, wenn explizit ausgelöst).
    
              
    

              
              
    
                
    Farbmarkierungen – Was bedeuten die Farben?
    
                
    AdminKit verwendet Farben um auf einen Blick zu signalisieren, wie kritisch ein Eintrag ist. Wichtig: die Farben basieren auf Heuristiken, nicht auf einer vollständigen Sicherheitsanalyse!
    
                
    
                  
    FarbeBedeutungGilt für
    
                  
    🟢 GrünKein Problem erkannt / Wert im normalen BereichStatus-Badges, Health Score, Scan-Zusammenfassung
    
                  
    🟡 Gelb (Zeile)Auffällig aber nicht kritisch: hoher CPU/RAM-Verbrauch, Drittanbieter-SoftwareProzesse (CPU >20% oder RAM >500MB), Autostart-Drittanbieter
    
                  
    🔴 Rot (Zeile)Heuristisch verdächtig: Pfad in Temp/Downloads, Root-Prozess aus Nicht-SystemverzeichnisProzesse und Autostart-Einträge mit verdächtigen Pfaden
    
                  
    ⚪ GrauUnbekannt / Keine Daten verfügbarSMART-Status, Hardware-Badge
    
                  
    Risiko-Score (0–100)Gewichteter Risikowert für Log-Ereignisse: 0–19 Rauschen, 20–59 mittleres Risiko, 60–100 hohes RisikoEreignis-Tab
    
                
    
                
    Was bedeutet „Grün" im Vollscan-Popup?
    
                
    Im Scan-Zusammenfassungs-Dialog stehen grüne Punkte für Bereiche ohne erkannte Probleme. Für den Autostart gilt: grün = weniger als 15 aktive Drittanbieter-Einträge. Das ist eine rein zahlenmäßige Schranke — AdminKit prüft dabei NICHT ob die Einträge sicher oder unsicher sind. Microsoft, PCVisit und andere bekannte Tools erscheinen also trotzdem grün, auch wenn sie Drittanbieter sind.
    
                
    Für eine echte Sicherheitsbewertung: Checkbox setzen → VirusTotal-Scan oder KI-Analyse nutzen.
    
              
    

              
              
    
                
    Dashboard
    
                
    Zentrale Übersicht des Systemzustands. Der Health Score (0–100%) bewertet das System anhand aller gesammelten Daten.
    
                
    
                  
    BadgeWas wird bewertet?Worauf achten?
    
                  
    HardwareCPU-Modell vorhanden, Kernerkennung„Kein CPU-Name" = Scan evtl. unvollständig
    
                  
    BetriebssystemOS-Name und VersionVeraltete Versionen = Sicherheitsrisiko
    
                  
    SMARTFestplatten-Gesundheit (Self-Monitoring)„CRITICAL" = Datenverlustgefahr, sofort Backup!
    
                  
    NetzwerkAdapter, offene Ports, FirewallViele offene Ports = Angriffsfläche
    
                
    
                
    Schnellaktionen
    
                
    Vordefinierte Reparaturschritte (DNS leeren, Netzwerk reparieren, Temp-Bereinigung). Jede Aktion zeigt einen Bestätigungs-Dialog — lesen, bevor ausgeführt wird! Einige Aktionen erfordern ggf. Neustart.
    
                
    Empfehlungen (Fixes)
    
                
    AdminKit erkennt häufige Konfigurationsprobleme und schlägt konkrete Fixes vor. Grüne Punkte = kein Handlungsbedarf, Orange/Rot = Aktion empfohlen.
    
              
    

              
              
    
                
    System-Tab
    
                
    Detaillierte Hardware- und Betriebssystem-Informationen.
    
                
    Hardware-Übersicht
    
                
    
                  
    FeldBedeutung
    
                  
    CPU / ChipProzessor-Modell. Bei Apple Silicon (M-Chips) steht hier z.B. „Apple M3 Pro".
    
                  
    Kerne / ThreadsPhysische Kerne und logische Threads. Mehr = schneller für Multitasking.
    
                  
    RAMArbeitsspeicher gesamt. Einzelne Slots zeigen Typ (LPDDR5), Takt und Hersteller.
    
                  
    MainboardHersteller, Modell und Seriennummer. Bei Macs: Modellbezeichnung.
    
                  
    GPUGrafikkarte(n). VRAM-Größe relevant für grafikintensive Anwendungen.
    
                
    
                
    SMART-Status
    
                
    Festplatten-Selbstdiagnose. Werte werden direkt von der Festplatten-Firmware gelesen.
    
                
    
                  
    StatusBedeutung & Aktion
    
                  
    ✅ OKFestplatte gesund. Kein Handlungsbedarf.
    
                  
    ⚠️ WARNINGErste Anzeichen von Verschleiß. Backup erstellen, Festplatte beobachten.
    
                  
    🔴 CRITICALFestplatte kurz vor Ausfall. Sofort Backup! Festplatte austauschen.
    
                  
    ⚪ UNKNOWNKein SMART-Zugriff (z.B. externe USB-Festplatten, NVMe ohne Treiber).
    
                
    
              
    

              
              
    
                
    Autostart
    
                
    Programme und Dienste, die beim Systemstart oder Login automatisch ausgeführt werden. Dies ist eine der wichtigsten Stellen für Sicherheitsanalysen — Malware versteckt sich oft hier.
    
                
    Typen-Erklärung (macOS)
    
                
    
                  
    TypWas ist das?SpeicherortWer kann es stoppen?
    
                  
    
                    LaunchAgent
                    Dienst/Programm das beim Login eines Benutzers startet. Läuft im Kontext des Benutzers (mit Benutzerrechten).
                    ~/Library/LaunchAgents/ (Benutzer)
    /Library/LaunchAgents/ (System, alle Nutzer)    		
                    Admin kann deaktivieren; Benutzer kann eigene deaktivieren
                  
    
                  
    
                    LaunchDaemon
                    Systemdienst der beim Systemstart startet, bevor sich ein Benutzer anmeldet. Läuft als Root.
                    /Library/LaunchDaemons/
    /System/Library/LaunchDaemons/ (Apple-intern)    		
                    Nur Admin; System-Daemons durch SIP geschützt
                  
    
                  
    
                    Login Item
                    App die beim Benutzer-Login startet (über macOS Systemeinstellungen). Neuerer Mechanismus für App Store Apps.
                    Systemeinstellungen → Allgemein → Anmeldeelemente
                    Benutzer kann über Systemeinstellungen deaktivieren
                  
    
                  
    
                    Cron-Job
                    Zeitgesteuerter Task (Unix-Standard). Läuft zu festgelegten Zeiten.
                    crontab -l im Terminal
                    Benutzer der den Cron erstellt hat
                  
    
                
    
                
    Was ist verdächtig?
    
                
      
                  
    • Einträge mit unleserlichen/zufälligen Namen (z.B. com.xjk93nq.agent)
      • 
                  
    • Pfade in Temp-Ordnern (/tmp/, ~/.cache/)
      • 
                  
    • LaunchDaemons von unbekannten Herstellern die als Root laufen
      • 
                  
    • Einträge ohne gültige Signatur (VirusTotal-Prüfung empfohlen)
      • 
                  
    • is_system: false + unbekannter Hersteller = genauer prüfen
      • 
                
    
                
    Deaktivieren / Aktivieren
    
                
    Über die ⏸/▶-Buttons in der Tabelle. Dabei wird launchctl unload -w / load -w ausgeführt. Der Eintrag bleibt erhalten, wird aber nicht mehr geladen. Bestätigungsdialog erscheint vor jeder Aktion.
    
              
    

              
              
    
                
    Dienste
    
                
    Systemdienste die dauerhaft im Hintergrund laufen (ähnlich wie Windows-Dienste). Auf macOS sind das hauptsächlich LaunchDaemons.
    
                
    Was sehe ich hier?
    
                
      
                  
    • Name / Label — Kennung des Dienstes (z.B. com.apple.mdmclient)
      • 
                  
    • Status — Läuft / Gestoppt / Deaktiviert
      • 
                  
    • PID — Prozess-ID wenn aktiv (0 = nicht laufend)
      • 
                  
    • Hersteller — Wer hat den Dienst installiert
      • 
                  
    • System — Apple-eigene Dienste vs. Drittanbieter
      • 
                
    
                
    Worauf achten?
    
                
      
                  
    • Drittanbieter-Dienste die dauerhaft laufen und hohen CPU/RAM verbrauchen
      • 
                  
    • Dienste mit unbekanntem Pfad oder aus temporären Verzeichnissen
      • 
                  
    • MDM-Dienste (mdmclient) zeigen an, dass das Gerät verwaltet wird
      • 
                  
    • Netzwerkdienste (z.B. com.openssh.sshd) nur wenn bewusst aktiviert
      • 
                
    
                
    Starten / Stoppen
    
                
    Über die Buttons in der Tabelle. Achtung: System-Dienste (is_system: true) können nicht dauerhaft deaktiviert werden — sie werden beim nächsten Neustart wiederhergestellt. Nur Drittanbieter-Dienste dauerhaft änderbar.
    
              
    

              
              
    
                
    Ereignisse (System-Log)
    
                
    Systemereignisse aus dem macOS Unified Log (oder Windows Ereignisprotokoll). Zeigt was auf dem System passiert ist — Fehler, Warnungen, Sicherheitsereignisse.
    
                
    Ereignis-Typen
    
                
    
                  
    TypBedeutungWann relevant?
    
                  
    🔴 ERROR / FAULTSchwerwiegende Fehler, Abstürze, fehlgeschlagene OperationenImmer untersuchen — zeigt echte Probleme
    
                  
    ⚠️ WARNINGWarnungen, nicht-kritische ProblemeHäufige Wiederholungen = Problem vorhanden
    
                  
    ℹ️ INFO / DEBUGNormale BetriebsmeldungenMeist ignorieren, bei Fehlersuche nützlich
    
                  
    🔒 AUTH / LOGINAn-/Abmeldungen, AuthentifizierungenUnerwartete Logins = Sicherheitsalarm!
    
                  
    🔋 KERNELBetriebssystem-Kern Ereignisse, TreiberfehlerKernel Panics, Hardware-Probleme
    
                
    
                
    Was macht die KI-Analyse?
    
                
    Markiere einzelne Ereignisse per Checkbox oder nutze „Alle mit KI analysieren". Die KI bewertet die Ereignisse und erklärt was passiert ist, ob etwas besorgniserregend ist und welche Schritte sinnvoll wären.
    
                
    Tipps zur Interpretation
    
                
      
                  
    • Zeitstempel — Wann trat das Ereignis auf? Korreliert mit Nutzerbeschwerden?
      • 
                  
    • Häufigkeit — Einmaliger Fehler oft harmlos; 100x in einer Stunde = Problem
      • 
                  
    • Quelle — Apple-eigene Quellen meist harmlos; unbekannte Prozesse genauer prüfen
      • 
                  
    • Meldungstext — Suchbegriffe in Google/VirusTotal recherchieren
      • 
                
    
              
    

              
              
    
                
    Prozesse
    
                
    Alle aktuell laufenden Prozesse mit CPU, RAM-Auslastung und Pfad. Ermöglicht das Erkennen von verdächtiger oder ressourcenfressender Software.
    
                
    Spalten-Erklärung
    
                
    
                  
    SpalteBedeutung
    
                  
    PIDProzess-ID (eindeutige Nummer, ändert sich nach Neustart)
    
                  
    NameProzessname (z.B. Google Chrome Helper)
    
                  
    CPU %Aktueller Prozessor-Anteil. Dauerhaft hohe Werte = Problem
    
                  
    RAM (MB)Arbeitsspeicher-Verbrauch. Hohe Werte bei vielen Prozessen = System langsam
    
                  
    PfadWo liegt die ausführbare Datei? Unbekannte Pfade = verdächtig
    
                  
    BenutzerUnter welchem Konto läuft der Prozess (root = hohe Rechte)
    
                
    
                
    Farbmarkierungen in der Prozesstabelle
    
                
    
                  
    FarbeBedeutungBedingung
    
                  
    🔴 Rote ZeileVerdächtiger Pfad — manuell prüfen!Prozess läuft aus /tmp/, /Downloads/, /var/folders/ oder AppData\Temp
    
                  
    🟡 Gelbe ZeileHohe RessourcennutzungCPU > 20% oder RAM > 500 MB
    
                  
    ⚪ NormalUnauffälligKein verdächtiger Pfad, moderate Auslastung
    
                
    
                
    Wichtig: Rote Zeilen sind Heuristiken — nicht jeder Prozess aus /tmp/ ist Malware (z.B. Systemdienste nutzen temporäre Pfade legitim). Immer im Kontext beurteilen und ggf. VT/KI-Analyse nutzen.
    
                
    Verdächtige Muster erkennen
    
                
      
                  
    • Prozesse aus /tmp/, /var/folders/, oder dem Downloads-Ordner → rot markiert
      • 
                  
    • Prozesse die als root laufen und nicht von Apple sind
      • 
                  
    • Viele identische Prozesse (z.B. 50x gleicher Name) = mögliche Endlosschleife oder Angriff
      • 
                  
    • Prozesse mit zufälligen Namen (z.B. aX7kQ9)
      • 
                  
    • Hoher CPU-Verbrauch ohne erkennbaren Grund → gelb markiert — Krypto-Mining oder Malware möglich
      • 
                
    
                
    Verdächtige Prozesse per Checkbox markieren und mit VT/KI analysieren lassen. Spalten durch Klick auf den Spaltenkopf sortieren.
    
              
    

              
              
    
                
    Netzwerk-Tab
    
                
    Adapter
    
                
    Alle Netzwerkschnittstellen (Ethernet, WiFi, VPN, Loopback). Zeigt IP-Adresse, MAC-Adresse und Status.
    
                
    Offene Ports
    
                
    Dienste die auf eingehende Verbindungen warten. Wichtig für Sicherheit:
    
                
    
                  
    PortTypischer DienstRisiko
    
                  
    22SSHHoch — nur wenn bewusst aktiviert
    
                  
    80/443HTTP/HTTPS (Webserver)Mittel — warum läuft ein Webserver?
    
                  
    548AFP (Apple Dateifreigabe)Mittel — Datenzugriff im Netz
    
                  
    3389RDP (Windows Fernzugriff)Hoch — häufiges Angriffsziel
    
                  
    5900VNC (Fernsteuerung)Hoch — nur mit Passwortschutz
    
                
    
                
    Aktive Verbindungen
    
                
    Zeigt laufende TCP/UDP-Verbindungen. Interessant: Verbindungen zu unbekannten IP-Adressen oder seltsame ausgehende Verbindungen.
    
                
    Netzwerk-Shares
    
                
    Freigegebene Ordner im Netzwerk. Nur was tatsächlich geteilt werden soll sollte hier erscheinen.
    
                
    ARP-Tabelle
    
                
    MAC-Adressen der Geräte im lokalen Netzwerk. Unbekannte Geräte im Netzwerk = Sicherheitscheck empfohlen.
    
              
    

              
              
    
                
    Software-Inventarisierung
    
                
    Alle installierten Anwendungen, Programme und Pakete. Wichtig für Lizenz-Management, Sicherheitsupdates und Compliance.
    
                
    Was sehe ich hier?
    
                
      
                  
    • Name & Version — Aktueller Versionsstand der Software
      • 
                  
    • Hersteller / Publisher — Wer hat die Software entwickelt
      • 
                  
    • Installationsdatum — Wann wurde die App installiert (hilfreich bei Problemsuche)
      • 
                  
    • Pfad — Wo ist die App installiert
      • 
                
    
                
    Worauf achten?
    
                
      
                  
    • Software mit sehr alten Versionen (mögliche Sicherheitslücken)
      • 
                  
    • Unbekannte Software die der Nutzer nicht kennt
      • 
                  
    • Mehrere Versionen derselben App (z.B. zwei Java-Versionen)
      • 
                  
    • Remote-Access-Tools (TeamViewer, AnyDesk) — nur wenn bewusst installiert
      • 
                  
    • P2P-Software (BitTorrent etc.) — mögliche Lizenz- und Sicherheitsprobleme
      • 
                
    
              
    

              
              
    
                
    Browser-Erweiterungen
    
                
    Installierte Erweiterungen in Chrome, Firefox, Safari und Edge. Erweiterungen haben weitreichende Zugriffsrechte auf besuchte Webseiten — daher kritisch für Sicherheit und Datenschutz.
    
                
    Was können Erweiterungen sehen/tun?
    
                
      
                  
    • Alle besuchten Webseiten lesen und modifizieren
      • 
                  
    • Passwörter und Formulardaten abfangen (bei schlechten Erweiterungen)
      • 
                  
    • Werbung einschleusen oder Seiten umleiten
      • 
                  
    • Suchergebnisse manipulieren
      • 
                
    
                
    Verdächtige Erweiterungen erkennen
    
                
      
                  
    • Erweiterungen die der Nutzer nicht kennt und nicht bewusst installiert hat
      • 
                  
    • Ad-Injector / Coupon-Apps (oft mit Datenweitergabe)
      • 
                  
    • Erweiterungen mit Namen wie „PDF Converter", „Video Downloader" von unbekanntem Hersteller
      • 
                  
    • Deaktivierte Erweiterungen mit erzwungenem Autostart (via Policy)
      • 
                  
    • Vergleich: Anzahl der Erweiterungen heute vs. letzter Scan
      • 
                
    
                
    Tipp: VT-Scan auf Erweiterungs-IDs und Pfade empfohlen bei Verdacht.
    
              
    

              
              
    
                
    USB-Geräte
    
                
    Alle aktuell angeschlossenen USB-Geräte. Wichtig für: Inventarisierung, Datensicherheit (unbekannte USB-Sticks), Hardware-Diagnose.
    
                
    Was sehe ich hier?
    
                
    
                  
    FeldBedeutung
    
                  
    Name / HerstellerProduktname und Hersteller-ID des Geräts
    
                  
    KlasseGerätetyp: Mass Storage, HID (Maus/Tastatur), Audio, Network, etc.
    
                  
    SeriennummerEindeutige Gerätekennung (hilfreich für Whitelist/Blacklist)
    
                  
    Vendor-ID / Product-IDHex-Codes für genaue Geräteerkennung
    
                
    
                
    Sicherheitshinweise
    
                
      
                  
    • Unbekannte USB-Speichergeräte = Datenverlust- oder Einschleusungsrisiko
      • 
                  
    • „BadUSB"-Angriffe: Ein USB-Stick täuscht vor eine Tastatur zu sein und führt Befehle aus
      • 
                  
    • In Unternehmensumgebungen: USB-Richtlinien (nur genehmigte Geräte) empfohlen
      • 
                  
    • Auswerfen-Button sicher trennen wenn Daten gerade geschrieben werden könnten
      • 
                
    
              
    

              
              
    
                
    Benutzerkonten
    
                
    Alle lokalen Benutzerkonten auf dem System. Kritisch für Sicherheitsaudits.
    
                
    Was sehe ich hier?
    
                
    
                  
    FeldBedeutung
    
                  
    Name / AnmeldenameBenutzername für Login
    
                  
    TypAdministrator, Standard, Gast, System-Account
    
                  
    AktiviertAktive vs. deaktivierte Konten
    
                  
    Letzter LoginWann war der Benutzer zuletzt aktiv
    
                  
    Passwort-ÄnderungWann wurde das Passwort zuletzt geändert
    
                
    
                
    Worauf achten?
    
                
      
                  
    • Zu viele Admins — Prinzip der minimalen Rechte: nur wer Admin-Rechte braucht, soll sie haben
      • 
                  
    • Inaktive Konten — Konten die nie oder lange nicht genutzt wurden sollten deaktiviert werden
      • 
                  
    • Gast-Account — Sollte in Unternehmensumgebungen deaktiviert sein
      • 
                  
    • Unbekannte Konten — System-Accounts von Software-Installationen vs. echte Personen unterscheiden
      • 
                  
    • Kein Passwort — Konten ohne Passwort sind ein gravierendes Sicherheitsrisiko
      • 
                
    
              
    

              
              
    
                
    Konfigurationsprofile (macOS)
    
                
    MDM-Profile (Mobile Device Management) und Konfigurationsprofile die Systemeinstellungen festlegen. Werden oft von IT-Abteilungen oder Software installiert.
    
                
    Was können Profile tun?
    
                
      
                  
    • Systemeinstellungen sperren (z.B. Firewall erzwingen)
      • 
                  
    • Zertifikate installieren (ermöglicht HTTPS-Entschlüsselung = Man-in-the-Middle)
      • 
                  
    • VPN- und WiFi-Verbindungen konfigurieren
      • 
                  
    • Apps installieren oder blockieren
      • 
                  
    • Screen Time / Sperren einrichten
      • 
                
    
                
    Worauf achten?
    
                
      
                  
    • Profile die nicht von der eigenen IT-Abteilung stammen
      • 
                  
    • Profile mit installierten Root-Zertifikaten von unbekannten Ausstellern — sehr gefährlich!
      • 
                  
    • Profile die unter „Nicht entfernbar" markiert sind (MDM-Kontrolle)
      • 
                  
    • Profile von Adware/Schadsoftware (oft als „Chrome Policy", „PDF Helper" etc.)
      • 
                
    
              
    

              
              
    
                
    VirusTotal
    
                
    Prüft Datei-Hashes gegen die VirusTotal-Datenbank (70+ Antiviren-Engines). Gibt schnellen Überblick ob eine Datei bekannte Schadsoftware ist.
    
                
    Einrichtung
    
                
      
                  
    1. Kostenlosen Account auf virustotal.com erstellen
      2. 
                  
    2. API-Key kopieren (Profil → API-Key)
      3. 
                  
    3. In AdminKit: Einstellungen → API-Schlüssel → VirusTotal eintragen
      4. 
                
    
                
    Wie funktioniert es?
    
                
    AdminKit berechnet den SHA256-Hash der Datei und fragt nur diesen Hash ab — die Datei selbst wird nie hochgeladen. Privacy-sicher für bekannte Software. Nur bei neuen/unbekannten Dateien (kein VT-Eintrag) ist ein Upload nötig.
    
                
    Ergebnisse interpretieren
    
                
    
                  
    ErgebnisBedeutung
    
                  
    ✅ Sauber0 von 70+ Engines schlagen Alarm. Wahrscheinlich sauber.
    
                  
    ⚠️ Verdächtig1–5 Engines. Könnte False Positive sein, genauer prüfen.
    
                  
    🔴 SchädlichMehrere Engines bestätigen Malware. Sofort handeln!
    
                  
    ⚪ Nicht gefundenUnbekannte Datei (evtl. neu/selbst kompiliert). Upload prüfen.
    
                
    
                
    Limit: Kostenloser API-Key: 4 Anfragen/Minute, 500/Tag. Premium für größere Scans empfohlen.

    Anbieter-Typen
    
                
    
                  
    AnbieterWie?DatenschutzKosten

      • 
                  
    • „Welche dieser Autostart-Einträge sollte ich deaktivieren?"
      • 
                  
    • „Gibt es in dieser Liste Anzeichen für Malware oder unerwünschte Software?"

    Vault & Sessions
    
                
    AdminKit speichert alle Daten lokal im Vault-Ordner neben der App-Datei.
    
                
    Vault-Speicherort
    
                
    
                  
    BetriebssystemStandard-Speicherort
    
                  
    macOS (App aus /Applications)~/adminkit_vault/ (im Home-Verzeichnis)
    
                  
    macOS (App auf USB-Stick)/Volumes/USB/adminkit_vault/ (neben der .app)
    
                  
    Windowsneben AdminKit.exe\adminkit_vault\
    
                
    
                
    Den aktuellen Vault-Pfad siehst du in der Statusleiste unten im App.
    
                
    Portabler Betrieb auf USB-Stick (empfohlen für Kunden-Scans)
    
                
      
                  
    1. AdminKit.app (oder .exe) auf USB-Stick kopieren
      2. 
                  
    2. Auf dem Kunden-Mac: App vom USB-Stick starten → Vault wird auf dem Stick erstellt
      3. 
                  
    3. Scan durchführen und Session speichern
      4. 
                  
    4. USB-Stick zum eigenen Mac mitnehmen
      5. 
                  
    5. AdminKit vom Stick starten (oder Vault-Ordner kopieren) → Session erscheint in der Session-Liste
      6. 
                  
    6. Ollama/LM Studio auf eigenem Mac laufen lassen → KI-Analyse der Kunden-Session durchführen
      7. 
                
    
                
    Hinweis macOS: Beim ersten Start auf einem fremden Mac erscheint ggf. eine Sicherheitswarnung (Gatekeeper). In Systemeinstellungen → Datenschutz & Sicherheit → „Trotzdem öffnen" klicken.
    
                
    Dateistruktur
    
                
      
                  
    • adminkit_vault/config.yaml — Einstellungen (Firmenname, API-Keys)
      • 
                  
    • adminkit_vault/<session>/system.json — Hardware- und OS-Scan
      • 
                  
    • adminkit_vault/<session>/network.json — Netzwerk-Scan
      • 
                  
    • adminkit_vault/<session>/software.json — Software-Inventar
      • 
                  
    • adminkit_vault/<session>/vt_audit.json — VirusTotal-Ergebnisse
      • 
                
    
                
    Datenschutz: Vault enthält sensible Systemdaten des Kunden. Nicht in iCloud/OneDrive synchronisieren, niemals in Git oder öffentliche Repositories pushen.
